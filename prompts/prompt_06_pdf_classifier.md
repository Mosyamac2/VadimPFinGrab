# Промпт 06. PDF Classifier

## Цель
Для каждого распакованного PDF определить: (а) является ли это формой отчётности и какой (МСФО/РСБУ/прочее), (б) машиночитаемый он или скан, (в) тип формы (баланс / ОФР / пояснения / прочее).

## Контекст из ТЗ
- Раздел 7.1, п.4: контракт стадии.
- Раздел 5.1: приоритеты типов отчётности (МСФО > РСБУ).
- Раздел 5.3: список метрик и их синонимов (используется для эвристик классификации).

## Задачи
1. Добавить зависимость `pymupdf` (fitz). `pdfplumber` понадобится позже, но можно поставить тоже.
2. Создать `src/edx/stages/classifier/`:
   - `pdf_inspector.py`:
     - `count_pages(path)`,
     - `extract_first_pages_text(path, pages: int = 3)` — через pymupdf,
     - `is_machine_readable(path)` — на первых N страницах суммарно > `min_text_chars` символов из конфига (`classifier.min_text_chars`, дефолт `400`); иначе считать сканом.
   - `heuristics.py`:
     - `detect_reporting_standard(text: str, metrics_config: MetricsConfig) -> Literal["IFRS","RSBU","OTHER"]`:
       - Признаки МСФО: «МСФО», «IFRS», «Consolidated», «Международными стандартами», «Группа», «Group».
       - Признаки РСБУ: «РСБУ», «бухгалтерский баланс», «Форма по ОКУД», «приказ Минфина».
       - При неоднозначности — сравнить количество совпадений; пустые показатели по обоим — `OTHER`.
     - `detect_report_form(text: str) -> Literal["balance_sheet","income_statement","cash_flow","notes","cover","other"]`:
       - простые регулярки (например, "Бухгалтерский баланс", "Отчёт о финансовых результатах", "Отчёт о движении денежных средств", "Statement of Financial Position", "Income Statement").
   - `service.py`: `ClassifierService.run(publication)`:
     - проходит по всем `documents` публикации;
     - для PDF (`mime_type` начинается с `application/pdf` или расширение `.pdf`):
       - заполняет `is_machine_readable`, `reporting_standard`, `report_form`, `page_count`;
     - для не-PDF — оставляет поля пустыми и помечает в логах.
     - При успехе — перевести публикацию в статус `classified`.
3. Поддержать применение **приоритетов из конфига** на этапе агрегации: если у публикации найдены и МСФО, и РСБУ за один период — ставить в `documents.is_primary_for_period` булев флаг (нужно расширить миграцию: `0002_classifier.sql` добавляет колонку `documents.is_primary_for_period INTEGER NOT NULL DEFAULT 0`).
   Конкретно: после классификации в публикации найти отчётный период (на этом этапе можно не извлекать — оставить заполнение `is_primary_for_period` на этап Metric Extractor; здесь миграция и колонка просто заводятся под будущее заполнение).
4. Логировать счётчики: сколько PDF, сколько машиночитаемых, сколько сканов, какие стандарты обнаружены.

## Тесты, которые должны проходить
- Юнит-тесты эвристик (`heuristics.py`) на синтетических кусках текста — оба стандарта, оба типа форм, неоднозначные кейсы.
- Юнит-тест `is_machine_readable`:
  - сгенерированный программно PDF с текстом → `True`;
  - PDF, состоящий из растрового изображения (использовать тестовую фикстуру `tests/fixtures/pdfs/scan.pdf`, ≤ 100 KB) → `False`.
- Юнит-тест `ClassifierService`:
  - на in-memory SQLite + временном каталоге с одним machine-readable PDF — после прогона запись в `documents` имеет ожидаемые значения.
- Все тесты не требуют сети и Tesseract.

## Definition of Done
- После стадии в `documents` заполнены `reporting_standard`, `is_machine_readable`, `report_form`, `page_count`.
- Публикация имеет статус `classified`.
- Эвристики вынесены в чистые функции и легко расширяются (добавление стандарта/формы — без правки сервисного слоя).
